Raspberry Pi 5
Setup Guide

M.2 HAT+ · Samsung 1TB NVMe · Argon NEO Case · SSD Boot

Your Hardware

Everything in this build

🍓

Raspberry Pi 5

4GB or 8GB — PCIe 2.0 x1 onboard

🔌

Official Pi 5 PSU

27W / 5V 5A USB-C — exactly what you need

📡

Raspberry Pi M.2 HAT+

Official HAT — PCIe FPC connector, M-key slot

💾

Samsung NVMe 1TB

M.2 M-key — fast, well supported on Pi 5

🧊

Argon NEO 5 Case

Passive + active cooling, fits M.2 HAT+ underneath

Step-by-Step Setup

Do these in order — some steps require a reboot

1

Start with a Working SD Card Boot

Before touching any hardware, make sure your Pi 5 boots normally from an SD card with a fresh Raspberry Pi OS (64-bit, Bookworm). You need this to update firmware and configure boot order. If you don't have one set up yet, flash the SD card first using Raspberry Pi Imager on any PC.

💡 Do not attach the M.2 HAT+ yet. Get the Pi running on SD first.
2

Update Everything & Flash the EEPROM

Boot from SD, open a terminal, and run:

sudo apt update && sudo apt full-upgrade -y
sudo rpi-eeprom-update -a
sudo reboot

After the reboot, verify the EEPROM is current:

sudo rpi-eeprom-update

Should say BOOTLOADER: up to date. This is required before NVMe boot will work reliably.

3

Set Boot Order to NVMe First

sudo raspi-config
  • Go to Advanced Options
  • Select Boot Order
  • Choose NVMe/USB Boot (puts NVMe before SD)
  • Finish & reboot (stay on SD for now)
💡 This just sets the order — Pi will still boot from SD if no NVMe is present.
4

Optionally Enable PCIe Gen 3 Optional

The official Pi 5 PCIe runs at Gen 2 by default (~450 MB/s). Gen 3 (~900 MB/s) is unofficial but works great with Samsung NVMe drives. To enable it:

sudo nano /boot/firmware/config.txt

Add this line at the bottom:

dtparam=pciex1_gen=3

Save with Ctrl+O, exit with Ctrl+X, then reboot.

⚠️ Gen 3 is not officially supported — some Pi units are more stable than others at Gen 3. If you get random crashes or boot issues, remove this line and stick with Gen 2.
5

Power Off & Attach the M.2 HAT+

Shut down completely — not just reboot:

sudo shutdown -h now

Unplug the power cord. Then:

  • Seat the Samsung NVMe SSD into the M-key M.2 slot on the HAT
  • Secure it with the included standoff screw
  • Connect the FPC ribbon cable between the HAT and the Pi 5's PCIe connector — lift the locking tab fully, push the cable all the way in, press the tab down to lock. Do both ends.
  • Stack the HAT onto the Pi's 40-pin GPIO header — press down firmly and evenly
⚠️ The FPC ribbon cable is the most fragile part. If the Pi won't detect the NVMe after boot, this cable is the first thing to reseat. Make sure both ends are fully locked.
6

Assemble the Argon NEO Case

The Argon NEO 5 case is designed to fit the Pi 5 with the M.2 HAT+ underneath. Assembly order:

  • Fit the Pi + HAT sandwich into the case base — HAT faces down into the case, Pi on top
  • Attach the fan header to the Pi's fan connector (small JST connector on the Pi 5 board)
  • Secure the case lid — screws go through the top plate into the standoffs
  • The NVMe SSD slot in the case aligns with the HAT below
💡 The Argon NEO has a built-in fan controller. After setup, install the Argon daemon to control fan speed via software.
curl -L https://download.argon40.com/argon-neo-5.sh | bash
7

Flash Raspberry Pi OS to the NVMe

Power the Pi back on — it will boot from SD since nothing is on the NVMe yet. Once booted, open a terminal and launch the imager:

sudo apt install rpi-imager -y
rpi-imager
  • Choose OS: Raspberry Pi OS (64-bit) — Bookworm
  • Choose Storage: your NVMe drive — shows as /dev/nvme0n1
  • Click the ⚙️ gear icon to pre-configure hostname, SSH, WiFi, and your username/password before flashing
  • Flash it
💡 Pre-configure SSH and your WiFi credentials in the imager's advanced settings — saves hassle after first NVMe boot.
8

Boot from NVMe

Remove the SD card, then power cycle the Pi. It should now boot directly from the Samsung NVMe.

Confirm you're running from NVMe:

lsblk

You should see nvme0n1 with partitions and your root / mounted on it. SD card should show nothing (it's removed).

# Also check read speed - Samsung should hit 400-900 MB/s
sudo hdparm -Tt /dev/nvme0n1
⚠️ If it boots to a blank screen or loops — reinsert the SD card, boot from it, and reseat the FPC cable on the HAT. That's the #1 cause of NVMe detection failure.
9

First-Time NVMe OS Setup

Once booted from NVMe, run a full update and configure your system:

sudo apt update && sudo apt full-upgrade -y

# Set hostname
sudo hostnamectl set-hostname pi

# Enable SSH if not already on
sudo systemctl enable --now ssh

# Check your IP
hostname -I

If you use SSH from other machines, add your key:

mkdir -p ~/.ssh
echo "YOUR_PUBLIC_KEY" >> ~/.ssh/authorized_keys
chmod 700 ~/.ssh && chmod 600 ~/.ssh/authorized_keys
10

Install Argon NEO Fan Daemon

The Argon NEO 5 case has a fan that needs software to control it. Without this, the fan either runs full speed or not at all:

curl -L https://download.argon40.com/argon-neo-5.sh | bash

This installs argonone-config. To adjust fan curve:

argonone-config

Default curve kicks the fan on around 55°C. You can tune it to stay quiet until higher temps.

💡 Check temps anytime with: vcgencmd measure_temp
11

Verify Everything is Working

# NVMe detected and mounted
lsblk

# PCIe device visible
lspci | grep -i samsung

# SSD speed test
sudo hdparm -Tt /dev/nvme0n1

# CPU temp
vcgencmd measure_temp

# Disk space (1TB should show ~930GB usable)
df -h /

# Uptime
uptime

If lspci shows your Samsung drive and df -h shows ~900GB free on /, you're fully set up.

Troubleshooting

Most common issues with this exact setup

🔌

Pi won't detect the NVMe at all

99% of the time this is the FPC ribbon cable. Power off, unplug, open the case, and reseat both ends of the flat cable. Make sure the locking tabs are fully closed. Also confirm the SSD standoff screw is tight.

🖥️

Blank screen / boot loop after removing SD

Either the NVMe wasn't flashed correctly, or boot order wasn't saved. Re-insert SD, boot, re-run sudo raspi-config to set boot order again, and verify the NVMe shows up in lsblk while booted from SD.

⚡

Random crashes / instability

If you enabled PCIe Gen 3, try removing dtparam=pciex1_gen=3 from /boot/firmware/config.txt. Gen 3 is unofficial — some Pi units don't like it. Gen 2 is stable and still fast (~450 MB/s).

🌡️

Pi running hot

Make sure the Argon daemon is installed and running (argonone-config). The NEO case is passive by default — the daemon activates the fan when temps rise. Check temp with vcgencmd measure_temp. Under 70°C is fine for the Pi 5.

📶

Can't SSH in after NVMe boot

SSH settings don't carry over automatically if you flashed a fresh OS image to NVMe. Either pre-configure SSH in Raspberry Pi Imager's advanced settings before flashing, or plug in a keyboard/monitor for first boot and run sudo systemctl enable --now ssh.

💾

Only 1 partition visible on NVMe

After first boot from NVMe, the OS auto-expands the filesystem to fill the drive. If it hasn't happened yet, run: sudo raspi-config → Advanced → Expand Filesystem. The full ~930GB should be available after a reboot.
