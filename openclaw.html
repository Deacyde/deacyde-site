OpenClaw Command Reference

Getting started

Use these first when you are checking health, onboarding a new setup, or trying to understand the current config.

openclaw help
Show top-level CLI help and available commands.
openclaw status
Show channel health and recent session recipients.
openclaw status --all
Run a fuller read-only diagnosis.
openclaw status --deep
Probe configured channels more deeply.
openclaw doctor
Run health checks and safe quick fixes.
openclaw doctor --fix
Apply recommended repairs automatically.
openclaw health
Fetch health from the running gateway.
openclaw setup
Initialize local config and agent workspace.
openclaw onboard
Interactive onboarding for gateway, workspace, and skills.
openclaw configure
Interactive configuration for credentials, channels, gateway, and defaults.
openclaw config *
Non-interactive config helpers like get, set, unset, file, and validate.
openclaw docs <query>
Search the live OpenClaw docs.
openclaw dashboard
Open the Control UI.
openclaw logs
Tail gateway file logs via RPC.

Gateway control

Use these when the service needs to be started, restarted, probed, or run in the foreground for debugging.

openclaw gateway status
Show gateway service status and probe reachability.
openclaw gateway start
Start the gateway service.
openclaw gateway stop
Stop the gateway service.
openclaw gateway restart
Restart the gateway service.
openclaw gateway health
Fetch gateway health.
openclaw gateway probe
Show reachability, discovery, health, and status summary.
openclaw gateway discover
Find local and wide-area gateway beacons.
openclaw gateway install
Install the gateway service.
openclaw gateway run
Run the gateway in the foreground.
openclaw gateway call <method>
Call a gateway RPC method directly.

Messages and channels

Use these when you are connecting chat services, sending messages, or working with channel-specific IDs and routing.

openclaw channels *
Manage connected chat channels like Discord, Telegram, or WhatsApp.
openclaw message *
Send, read, and manage messages.
openclaw message send --target <target> --message "..."
Send a message through a configured channel.
openclaw agent
Run one agent turn via the gateway.
openclaw directory *
Look up contact and group IDs for supported channels.

Agents, memory, and sessions

Use these for agent creation, routing, memory inspection, model management, and checking stored sessions or tasks.

openclaw agents *
Manage isolated agents, workspaces, auth, and routing.
openclaw sessions *
List stored conversation sessions.
openclaw memory
Search, inspect, and reindex memory files.
openclaw skills *
List and inspect available skills.
openclaw models *
Discover, scan, and configure models.
openclaw tasks *
Inspect durable background task state.

Devices and nodes

Use these for pairing hardware, managing nodes, and handling QR-based device linking.

openclaw devices *
Device pairing and token management.
openclaw nodes *
Manage gateway-owned node pairing and commands.
openclaw node *
Run and manage the headless node host service.
openclaw pairing *
Approve inbound secure DM pairing requests.
openclaw qr
Generate a mobile pairing QR/setup code.

Automation

Use these when you want the system doing work on its own through schedules, hooks, plugins, or webhook-driven actions.

openclaw cron *
Manage cron jobs via the gateway scheduler.
openclaw system *
System events, heartbeat, and presence.
openclaw hooks *
Manage internal agent hooks.
openclaw plugins *
Manage OpenClaw plugins and extensions.
openclaw webhooks *
Webhook helpers and integrations.

Security and maintenance

Use these for audits, updates, backups, approvals, and cleanup.

openclaw security *
Security tools and local config audits.
openclaw update *
Update OpenClaw and inspect update channel status.
openclaw backup *
Create and verify local backup archives.
openclaw approvals *
Manage exec approvals.
openclaw exec-policy *
Show or sync requested exec policy.
openclaw reset
Reset local config and state while keeping the CLI installed.
openclaw uninstall
Uninstall the gateway service and local data while leaving the CLI installed.
